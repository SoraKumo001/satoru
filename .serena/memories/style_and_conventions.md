# Style and Conventions

## Code Style
- **C++**: Adheres to `.clang-format`. Use `pnpm format` to apply.
- **TypeScript/JavaScript**: Uses Prettier. Use `pnpm format` to apply.
- **Naming**: Follow existing project patterns (mimic surrounding code).
- **Comments**: Focus on "why" rather than "what". Avoid excessive chitchat.

## Environment & Tooling
- **OS**: Windows (PowerShell).
- **Shell Commands**:
  - Use `;` for sequential execution (e.g., `cmd1 ; cmd2`).
  - Do NOT use `&&` or `||`.
  - Do NOT execute multiple shell commands in parallel.
- **Git**: 
  - Do not `add`, `commit`, or `push` unless explicitly asked.
  - Use `status`, `diff`, and `log` for context.
